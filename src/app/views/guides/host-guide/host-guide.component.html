Hosting Guide

1 Click the "host session" Button

To host a session, click the "host session" button at the right side of the navigation bar.

2 Enter the session details

Only the Dodo Code is required to start a session, other details such as title or description are optional. You can change all settings after starting the session, so don't sweat the details.

The possible settings are as follows:

  • Dodo Code: the Dodo code, required to start the session.
  • Title: a short description to summarize what the session is for, e.g. "REDD", "CELESTE".
  • Description: a longer description for all the details that didn't fit into the title.
  • Turnip Prices: the prices turnips sell for / cost on your island. Leave at 0 if you don't care.
  • unlisted: whether or not the session is publically visible in the Session List or not. Private hosting should probably happen over Friend Codes instead of on this site, but it's an option.
  • public requester count: whether to publically display the number of people who you accepted to join and how many are still requesting to join. As the host, you always see these numbers.
  • verified-only: whether only verified players may request to join the session or if any player can request to join.
  • auto-accept verified players: whether verified players are automatically accepted and are given access to the Dodo Code of the session. Use in conjunction with verified-only for AFK hosting.

3 Do the Hosting

Congratulations, after entering the session details and clicking "create session", you have started hosting.

After starting the session, you can obtain copy the url to the session to share with others by clicking the clipboard symbol next to the session ID.

You can also edit session details by clicking the pencil symbols next to the fields, or the "Show / Edit Dodo Code" button to show or edit the Dodo Code.

To change session status, click the pencil symbol next to the session status and select the desired status.

To edit session settings like unlisted, verified-only, etc. click the cog in the top right corner of the session. This opens up the session settings dialog.

4 Getting Requesters

Once the session is started, other players can request to join. When a player requests to join, they appear in the list below the session.

The list is updated automatically when a player requests to join, you do not need to refresh the page to update!

Now you are presented with the option to accept the player and give them permission to view the Dodo Code, or reject them and prevent them from requesting to join this session again.

You can also click the to block a player, preventing the player from requesting to join any of your sessions.

Now that the player is accepted they may obtain the Dodo Code.

A more visible next to the player ID means that the player has obtained the Dodo Code. If the plane stays transparent, it means the player has yet to obtain the Dodo Code.

You may also revoke the permission to obtain the Dodo Code from the player by clicking "widthdraw accept" (the same goes for reject), causing them to disappear from the list and they now have to request to join again.
